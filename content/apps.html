Word Virus

Ongoing project, 1% complete. This will be a web-based notepad app with a focus on creative writing functionality. It has or will have a number of features that are rarely (if ever?) seen in typical notepads. There will be various generator options to create random pieces of text, randomization options to jumble existing text in various ways, and other special options such as invisible text and disabling of delete/backspace. While the UI is still ugly/buggy and only a small percentage of the functions have been implemented (also buggy!), I wanted to put this here. I will update as I progress. I intend for this to be my first serious big project, and think that the 0.001% of internet users that might ever use something odd like this might really like it. Try the current iteration.

Unrealistic Life Simulator

This began as the Chapter 7 project "Electronic Life" in Eloquent JavaScript. It's actually older than some of the projects below, but I was holding off on putting it here because I had hoped to improve it more...but sadly I have not been able to make time for it, so here it is, 99% complete. Anyway, this is by far my favorite project I've made in JavaScript. The original project from the book just had a simple text-based display. After completing that, I went ahead and converted the JS DOM code to use jQuery (it was the only way I knew how to work with the DOM at the time), then I improved the display quite a bit, added functionality to change the speed of the simulation, various options for editing the map, and perhaps most importantly I came up with a number of my own lifeforms to add into the program. Play it.

Primary issue: I have not gotten around to implementing the save/load functionality yet.

Map Quiz

One of the more recent little web apps I made. It uses the Google Maps API to load a satellite photo of a city (randomly chosen from the 22 I currently have in the code). You need to choose the right city name based only on the satellite photo. It runs through all 22 cities then tells you how many you got right at the end. Play it.

Known issues/future plans: The buttons need to be disabled while showing the right/wrong message, the buttons should be disabled at the end of the game, and the game should give an option to restart instead of just stopping completely at the end. Also, I would like to add functionality to pull city names and coordinates from an online API instead of storing the info in the code by hand.

Snake Clone

Began as a snake clone, but evolved into something a little different based solely on the images I ended up using. As usual, movement is automatic and you change direction with the arrow keys. You eat to grow, but in this game you can cross your own path. The hazards come in the form of the spikes that are added to the game each time you "level up." Stars give bonus points (level times length) and your high scores are saved in localStorage. Play it.

Known issues/future plans: Usual cleanup and improvements. Maybe some tweaks to the gameplay. And even if the display scales correctly on smaller devices (which I have yet to address), arrow keys are not touchscreen friendly.

Memory

Pretty typical memory matching game where you flip cards to match the letters of the alphabet. To add a unique element and extra programming challenge, I added two special cards. The asterisk randomly shuffles the cards if you pick it. The plus sign auto-matches a pair for you if you pick it. (Both special cards flip back over if the cards are reshuffled). Play it.

Known issues/future plans: The usual cleanups and improvements, but it plays fine for now so that's why it's here.

Lights Out

Typical click-to-shut-off-all-the-lights type of game. Though with various game modes to make it a more interesting challenge. This was my third ever JavaScript project and my first that was self-directed rather than being an assignment/tutorial. Play it.

Known issues/future plans: The code could use some refactoring and the display could be far more responsive and compatible. Also I wanted to improve the UI and visuals in general.
